# Code of Conduct

## Our Pledge

We are committed to keeping FEZrs open, collaborative, and welcoming for everyone.
All contributors, maintainers, and community members are expected to communicate respectfully and work together professionally.

## Our Standards

Examples of behavior that contributes to a positive environment include:

- Writing clean, maintainable, and well-structured code
- Creating Pull Requests (PRs) for all changes
- Pulling the latest changes from `main` and `dev` before starting new work
- Developing new features from the `dev` branch and merging back into `dev`
- Adding tests for new features and major functionality changes
- Reviewing AI-generated code carefully before submission
- Respecting feedback and participating in constructive discussions

Examples of unacceptable behavior include:

- Toxic, insulting, or disrespectful communication
- Spam, low-quality, or intentionally harmful contributions
- Repeatedly submitting unreviewed or broken code
- Plagiarism or misuse of other contributors' work
- Any behavior that negatively affects the project or community

## Our Responsibilities

Project maintainers are responsible for reviewing contributions and maintaining the quality, stability, and direction of the project.

Maintainers may reject, edit, or remove contributions that do not follow this Code of Conduct or the project's development standards.

## Scope

This Code of Conduct applies to all FEZrs project spaces, including GitHub issues, pull requests, discussions, commits, and other community interactions related to the project.

## Enforcement

Instances of unacceptable behavior may result in warnings, rejected contributions, temporary restrictions, or permanent removal from the project community, depending on the severity of the violation.

## Attribution

This Code of Conduct is inspired by open-source community standards including the Contributor Covenant and practices used by major open-source projects.
