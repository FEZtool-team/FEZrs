# Security Policy

FEZrs is a scientific Python library for remote sensing, image processing, feature extraction, and geospatial raster analysis. Security reports help protect users, research workflows, and downstream projects that rely on FEZrs.

## Reporting a Vulnerability

Please report suspected vulnerabilities by email:

team@feztool.com

Do not open a public GitHub issue for sensitive security problems. If the issue is already public or was discovered in a public place, please still email the team and include the public link.

## What to Include

Please include enough evidence for the team to understand and reproduce the issue:

- A clear summary of the vulnerability.
- The affected FEZrs version, commit, branch, or release.
- Where you saw the issue, such as a file path, function name, command output, GitHub issue, advisory, dependency report, or external page.
- Steps to reproduce the problem.
- Logs, traceback output, screenshots, proof-of-concept details, or sample input data when safe to share.
- Expected behavior and actual behavior.
- Any known impact, such as data exposure, unsafe file handling, dependency risk, denial of service, or incorrect scientific output caused by malicious input.
- Your contact information for follow-up questions.

Please avoid sending private credentials, tokens, or sensitive datasets. If a reproduction case needs sensitive data, describe the shape of the data and the minimum conditions needed to reproduce the issue.

## Response

The FEZtool team will review reports sent to team@feztool.com and may contact you for clarification, additional logs, or reproduction details.

Once the issue is understood, the team will decide whether it requires a security fix, documentation update, dependency update, or another mitigation.

## Responsible Disclosure

Please give the team reasonable time to investigate and prepare a fix before sharing sensitive details publicly. Public reports should avoid exploit instructions until a fix or mitigation is available.
